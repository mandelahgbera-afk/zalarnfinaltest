7. RLS SECURITY VERIFICATION
=============================
Test Query (Run in Supabase SQL Editor as user1@example.com):
  SELECT * FROM public.user_balances;
  Result: ONLY sees own balance row

Test Query (Run in Supabase SQL Editor as admin@example.com):
  SELECT * FROM public.user_balances;
  Result: SEES ALL balance rows

This proves RLS is working:
- User: "balances_own" policy restricts to auth_email()
- Admin: "balances_admin_all" policy allows is_admin() = true

Privilege Escalation Prevention:
- User cannot UPDATE their own role field (RLS policy prevents)
- User cannot UPDATE balance (no INSERT/UPDATE policy for users)
- Admins can UPDATE any field via "admin_all" policies
- All policies use is_admin() function which queries users table
